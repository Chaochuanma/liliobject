## 路線 A:Cloudflare(推薦 — 網域+空間+DNS 同一處,最省事最便宜)

**買網域**
1. 註冊 https://dash.cloudflare.com 帳號。
2. 左側選單 **Domain Registration → Register Domains**,搜尋 `liliobject.com`,結帳(約 US$10/年,接近成本價)。

**放網站(Cloudflare Pages)**
3. 左側 **Workers & Pages → Create → Pages → Upload assets**。
4. 專案名打 `liliobject`,把這個資料夾裡的 **`index.html`** 拖上去,Deploy。
5. 部署完會給你一個 `liliobject.pages.dev` 網址,先確認畫面正常。

**接上你的網域**
6. 在該 Pages 專案 → **Custom domains → Set up a custom domain** → 輸入 `liliobject.com`(再加一個 `www.liliobject.com`)。
7. 因為網域也在 Cloudflare,DNS 會**自動設定**,幾分鐘到數十分鐘生效。完成。

---

## 路線 B:最簡單、不用任何指令(Netlify 拖檔)

1. 開 https://app.netlify.com/drop ,把這個資料夾**整個拖進去** → 立刻有一個臨時網址。
2. 註冊/登入 Netlify 保存這個站。
3. **Domains → Add a domain** 輸入 `liliobject.com`(網域要先在別處買好,例如 Cloudflare Registrar 或 Namecheap)。
4. Netlify 會給你幾筆 DNS 記錄,到你買網域的地方填進去即可。

> 路線 B 的網域和空間在不同平台,DNS 要手動接;路線 A 全在一處、最不容易出錯,**建議用 A**。

## 路線 C:GitHub Pages（已幫你設好,免費)

這個資料夾**已經是 git repo**,並含 `CNAME`(liliobject.com)與 `.nojekyll`。只差「建遠端 repo + 推上去 + 開 Pages + 設 DNS」。

**1. 在 GitHub 建一個 repo**
到 https://github.com/new ,Repository name 例如 `liliobject`,設 Public,**不要**勾 Add README,Create。

**2. 推上去**(把 `你的帳號` 換成你的 GitHub 帳號)
```bash
cd /Users/peterchen/lili-site
git remote add origin https://github.com/你的帳號/liliobject.git
git push -u origin main
```

**3. 開啟 Pages**
repo → **Settings → Pages** → Source 選 **Deploy from a branch** → Branch 選 **main** / **/(root)** → Save。
等 1–2 分鐘,會出現 `https://你的帳號.github.io/liliobject/`(或因 CNAME 直接顯示自訂網域)。

**4. 設 DNS(在你買 liliobject.com 的註冊商後台填)**

| 類型 | 主機/名稱 | 值 |
| --- | --- | --- |
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |
| CNAME | www | 你的帳號.github.io |

(可選 IPv6 AAAA:`2606:50c0:8000::153`、`8001::153`、`8002::153`、`8003::153`)

**5. 回 Settings → Pages**,Custom domain 應已自動帶入 `liliobject.com`(來自 CNAME 檔);DNS 生效後勾 **Enforce HTTPS**。完成 ✅

> DNS 生效從幾分鐘到數小時不等。
